### 五、后端 API 实现

**时间**：中午-下午

**路由文件**：

| 文件 | 路由 | 功能 |
|------|------|------|
| `routers/auth.py` | `/api/auth/*` | 登录/登出/管理员注册用户/用户管理 |
| `routers/projects.py` | `/api/projects/*` | 项目 CRUD（全部受 auth 保护） |
| `routers/scripts.py` | `/api/projects/{id}/scripts/*` | AI脚本生成 + AI改写 |
| `routers/modules.py` | `/api/projects/{id}/storyboard/*` | 分镜生成 |
|  | `/api/projects/{id}/images/*` | 分镜图生成 |
|  | `/api/projects/{id}/video/*` | 视频合成 |
|  | `/api/ai-platform/*` | AI平台配置 CRUD |

**核心服务**：

| 服务 | 文件 | 功能 |
|------|------|------|
| AI 代理层 | `services/ai_proxy.py` | 根据用户配置路由到 Codex 或 豆包 |
| JWT 鉴权 | `utils/auth.py` | HMAC-SHA256 token + pbkdf2 密码哈希 |
| 鉴权中间件 | `utils/auth_deps.py` | get_current_user / require_admin |
| 加密工具 | `utils/crypto.py` | AES-256-GCM API Key 加解密 |

**关键实现细节**：
- AI 代理层统一接口：`generate_script()` / `generate_storyboard()` / `generate_image_prompt()` / `optimize_video_script()`
- 所有 AI 调用通过 httpx 异步客户端，120s 超时
- API Key 加密存储，密钥通过环境变量注入
- 管理员初始化：首次启动自动创建 admin/admin123

### 七、认证与权限

**时间**：下午

**实现方案**：
- 管理员：内置 admin/admin123，系统启动自动创建
- 用户：管理员在「用户管理」页面创建账号
- 密码：pbkdf2_hmac (sha256, 100000 iterations, 16-byte salt)
- Token：HMAC-SHA256 JWT 风格，24h 过期
- 中间件：FastAPI Depends 注入，`get_current_user` / `require_admin`
- 前端：Zustand store + localStorage 持久化 + 自动恢复登录态

**用户隔离**：
- 所有 API 基于 `user_id` 过滤数据
- 用户只能看到自己的项目
- 管理员可以管理所有用户
