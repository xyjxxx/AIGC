## 一、项目概述

**微型一站式AI创作带货视频平台** 是一款面向中小电商卖家的零门槛AI视频创作工具。用户上传商品素材后，平台自动完成从脚本撰写、分镜设计、分镜图生成到成品视频输出的全链路创作。

### 核心价值主张

> **"上传商品素材，一键生成专业带货视频"**

| 维度 | 价值 |
|------|------|
| **省时** | 从几天缩短到几分钟 |
| **省力** | 无需文案、拍摄、剪辑技能 |
| **省钱** | 单条视频成本趋近于零 |
| **专业** | AI理解商品卖点，生成营销级别内容 |

### 技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| AI 平台 | OpenAI Codex / 字节豆包 | 用户自选，自带账号，费用自理 |
| 数据库 | MySQL 8.0 | 国内生态成熟，OLTP 性能优秀 |
| 存储 | 本地磁盘 (NAS) | 数据不出域，适合私有化部署 |
| 前端 | Next.js 16 + React 18 | 现代化 SSR 框架 |
| 后端 | Python FastAPI | AI 生态兼容，异步高性能 |

## 三、功能模块设计

### 模块总览

```
素材输入 ──→ 智能脚本生成 ──→ 分镜画板设计 ──→ 分镜图生成 ──→ 视频合成输出
 (Step 0)      (Step 1)        (Step 2)        (Step 3)       (Step 4)
```

### 模块一：智能脚本生成

**6 种模板**：
- 痛点解决型 (30-60s) — 家居/个护
- 测评对比型 (45-90s) — 数码/美妆
- 使用教程型 (60-120s) — 小家电/DIY
- 场景种草型 (15-45s) — 服饰/香薰
- 口播种草型 (30-60s) — 全品类
- 促销快闪型 (15-30s) — 大促/爆品

**核心功能**：
- 5种素材入口（链接/图片/手动/文档/CSV）
- AI 一次生成 3 个版本并排对比
- 段落增删改 + 拖拽排序 + AI 改写
- 广告法违禁词自动检测

### 模块二：分镜画板设计

- 脚本段落 → AI 自动拆解分镜
- 可视化时间线拖拽编辑
- 分镜卡包含：编号/时长/景别/运镜/画面描述/口播/转场
- 智能建议：节奏检测 + 景别配比 + 转场推荐

### 模块三：分镜图生成

- 6种图像风格（真实摄影/3D渲染/扁平插画/国潮风/极简白底/动漫风）
- 自动提示词构建引擎
- 角色一致性保持
- 图片画廊 + 变体选择 + 局部重绘
- 使用用户自选的 AI 平台（Codex/豆包）

### 模块四：视频合成输出

- TTS 配音 (40+ 音色，含方言)
- 自动字幕 + 关键词高亮
- BGM 曲库 + AI 生成 + 智能闪避
- 多平台预设输出（抖音/小红书/B站/视频号）
- 1080P/4K，多种比例

## 四、UI-UX 设计方案

### 设计理念：「丝滑流水线」

```
素材 ▸ 脚本 ▸ 分镜 ▸ 生图 ▸ 成片
  └── 一气呵成，每步可回溯 ────┘
```

### 设计系统

**色彩**：
| Token | 色值 | 用途 |
|-------|------|------|
| 主背景 | #0A0A0F | 暗色沉浸 |
| 卡片层 | #14141F | 玻璃质感 |
| 强调色 | #6C5CE7 | AI 紫色 |
| AI 渐变 | #6C5CE7→#00D2FF | 品牌标识 |

**字体**：Inter（正文）+ JetBrains Mono（代码/数据）

**核心组件**：
- 6 种按钮（Primary/Secondary/Ghost/Danger/Success/AI-Gradient）
- 玻璃质感卡片（backdrop-filter 磨砂）
- AI 进度条（渐变流光动画）
- 步骤条导航（呼吸光晕指示器）
- 时间线分镜卡片（可拖拽 + 选中态）

### 微交互动画

| 交互 | 触发 | 动画 |
|------|------|------|
| 分镜拖拽 | Drag | 卡片升起 + 阴影 + 紫色描边 |
| AI 生成 | 任务完成 | Bounce-In + 音效 |
| 步骤切换 | 点击 | 横向滑动 300ms |
| 导出按钮 | Hover | 渐变色流光 800ms |

### 响应式策略

| 分辨率 | 布局 |
|--------|------|
| >1280px | 侧栏 + 宽画板 + 并排对比 |
| 768-1280px | 顶部导航 + 双版对比 |
| <768px | 底部Tab + 单版滑动 |

## 六、数据库设计

### ER 图

```
users ──1:N── projects ──1:N── scripts ──1:N── script_segments
  │                │
  │                ├──1:N── storyboard_shots ──1:N── shot_images
  │                │
  │                └──1:1── videos
  │
  └──1:N── user_ai_configs
```

### 核心表结构

**users** — 用户表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| username | VARCHAR(64) UNIQUE | 登录用户名 |
| password_hash | VARCHAR(256) | pbkdf2 哈希 |
| role | VARCHAR(10) | admin / user |
| is_active | BOOLEAN | 管理员可禁用 |

**projects** — 项目表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 自增主键 |
| user_id | BIGINT FK→users | 所属用户 |
| name | VARCHAR(200) | 项目名称 |
| status | VARCHAR(20) | draft→script→storyboard→images→video→done |
| current_step | INT | 0-4 步骤指示 |
| ai_platform | VARCHAR(20) | 使用的AI平台 |

**scripts** — 脚本表（多版本）
**script_segments** — 脚本段落表
**storyboard_shots** — 分镜表
**shot_images** — 分镜图表
**videos** — 视频表
**user_ai_configs** — 用户AI平台配置（加密存储）

## 七、权限与安全

### 认证流程

```
管理员登录 → 创建用户 → 用户获得账号密码 → 用户登录 → 获得 JWT Token
                                                    ↓
                                           所有 API 请求携带 Token
                                                    ↓
                                          服务端验证 → 返回数据
```

### 安全措施

| 措施 | 实现 |
|------|------|
| 密码存储 | pbkdf2_hmac (sha256, 100000 iterations) |
| API Token | AES-256-GCM 加密存储，密钥环境变量注入 |
| 传输安全 | HTTPS（生产环境） |
| JWT 令牌 | HMAC-SHA256 签名，24h 过期 |
| 权限控制 | FastAPI 依赖注入中间件 |
| 用户隔离 | 所有查询基于 user_id 过滤 |

## 九、部署与启动

### 环境要求

| 软件 | 版本 | 说明 |
|------|------|------|
| Node.js | ≥18 | npm 包管理 |
| Python | ≥3.11 | 后端运行 |
| Docker Desktop | 最新 | MySQL + Redis 容器 |
| Windows/Mac/Linux | — | 开发环境 |

### 安装步骤

```bash
# 1. 克隆 / 进入项目
cd shipin-gongfang

# 2. 启动 Docker Desktop（手动启动）
#    等待 Docker 图标变绿

# 3. 启动数据库
docker-compose up -d

# 4. 安装后端依赖并启动
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --reload --port 8000

# 5. 安装前端依赖并启动（新终端）
cd frontend
npm install
npm run dev

# 6. 访问
# 前端: http://localhost:3000
# 后端API文档: http://localhost:8000/docs
# 管理员: admin / admin123
```

## 十二、风险评估

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| AI 生成质量不稳定 | 用户体验差 | 多模型冗余 + 用户手动微调 + 持续优化 |
| AI API 服务中断 | 功能不可用 | 双平台切换（Codex↔豆包）+ 降级提示 |
| 版权纠纷 | 法律风险 | 免版权素材库 + AI 生成原创性确保 + 免责条款 |
| 同质化竞争 | 市场份额 | 垂直深耕带货场景 + 模板生态壁垒 |
| API Key 泄露 | 安全事件 | AES-256 加密 + HTTPS + 不存明文 |
| 内容合规 | 平台处罚 | 广告法检测 + 敏感词过滤 |
